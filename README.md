# Time Flies Bot

Slack 工時紀錄機器人，讓你透過 Slash Commands 快速記錄每日工時，並自動將日報寫入 Slack Canvas「我愛工作」。

---

## 指令說明

### `/start` — 開始計時

開啟選單，選擇專案與工作內容，Bot 開始計時。

**流程：**
1. 輸入 `/start`
2. 選擇專案（Richart / ASUS / BOT）
3. 填入工作內容
4. 送出後 Bot 回覆：`▶ 已開始計時：[Richart] 開會討論需求`

> 若已有進行中的計時，Bot 會提示先執行 `/stop`。

---

### `/stop` — 停止計時並儲存

停止目前計時，自動計算時數（四捨五入至 0.5 小時）並儲存。

**流程：**
1. 輸入 `/stop`
2. Bot 回覆：`⏹ 已記錄：[Richart] 開會討論需求 — 1.5 小時`

> 若沒有進行中的計時，Bot 會提示先執行 `/start`。

---

### `/log` — 手動補記工時

不使用計時器，直接輸入時數補記。

**流程：**
1. 輸入 `/log`
2. 選擇專案、填入工作內容與時數
3. 送出後 Bot 回覆確認，並顯示兩個按鈕：
   - **再新增一筆** — 重新開啟表單繼續記錄
   - **完成** — 結束

---

### `/summary` — 產生工時日報

將指定區間的工時格式化後寫入 Canvas「我愛工作」。

| 用法 | 說明 |
|------|------|
| `/summary` | 當天 |
| `/summary 2026-04-14` | 指定單日 |
| `/summary 2026-04-01 2026-04-14` | 指定區間 |
| `/summary this-week` | 本週（週一～今天） |
| `/summary last-week` | 上週（週一～週日） |

寫入完成後 Bot 回覆：`已將 2026-04-14 紀錄存入 Canvas ✓`

> 若指定區間無任何紀錄，Bot 會提示「該區間沒有工時紀錄。」

---

## 自動排程

Bot 會在以下時間點自動將當日工時寫入 Canvas（**週一至週五**，台灣時間）：

- 09:00
- 14:00
- 19:00

**當天有紀錄：** 自動 append 至 Canvas，並傳送確認訊息。  
**當天無紀錄：** 在 DM 發送提醒：`你今天還沒有工時紀錄，記得填寫！`

---

## Canvas 格式範例

```
## 2026-04-14
### Richart:
- 開會討論需求 1.5 小時
- 撰寫規格 2 小時

### ASUS:
- UI 調整 1 小時
```

## 多人使用

同一個 Workspace 的所有成員皆可使用，每位使用者的紀錄完全獨立，互不影響。
