# 工時紀錄 Slack Bot — 設計文件

**日期**：2026-04-14  
**狀態**：已確認

---

## 概述

一個部署在 Slack Workspace 的 Bot，讓使用者透過 Slash Commands 紀錄每日工時，最終將格式化的日報 append 到 Slack Canvas 中保存。

初期供個人使用，架構支援同 Workspace 多人同時使用。

---

## 技術棧

- **執行環境**：Node.js
- **Slack SDK**：`@slack/bolt`
- **資料儲存**：
  - 進行中計時器：Server 記憶體（Map，以 `userId` 為 key）
  - 工時條目：`data/logs.json`（每次 `/stop` 或 `/log` 後即時寫入，支援跨日查詢）
  - 最終日報：Slack Canvas，名稱為「我愛工作」（每次 `/summary` 或排程自動寫入時 append）
  - 寫入前檢查 DM 頻道是否已有「我愛工作」Canvas；不存在則自動建立

**執行環境**：Railway（雲端，24 小時運行，不依賴本機 Mac）  
**持久化儲存**：Railway Persistent Volume，掛載路徑 `/data`，`logs.json` 存於其中，容器重啟後資料不遺失

`logs.json` 結構：
```json
{
  "U12345678": {
    "2026-04-14": [
      { "project": "RC", "content": "開會討論需求", "hours": 1.5 }
    ]
  }
}
```

## 指令設計

### `/start`

**用途**：開始計時一項工作

**互動流程**：
1. 開啟 Modal，包含：
   - 下拉選單：選擇專案（RC / ASUS / BOT）
   - 文字欄位：工作內容（必填）
2. 送出後 Bot 回覆：`▶ 已開始計時：[RC] 開會討論需求`
3. 記憶體儲存：`userId → { project, content, startTime }`

**邊界條件**：
- 若已有進行中的計時，Bot 提示：`你有一筆尚未停止的計時（[RC] 開會討論需求），請先 /stop 後再開始新的。`

---

### `/stop`

**用途**：停止計時並儲存本筆工時

**互動流程**：
1. 從記憶體取得當前計時資料
2. 計算時數：`(now - startTime)`，四捨五入至 0.5 小時
3. 將條目加入當日 list：`{ project, content, hours, date }`
4. Bot 回覆：`⏹ 已記錄：[RC] 開會討論需求 — 1.5 小時`

**邊界條件**：
- 若無進行中的計時，Bot 提示：`目前沒有進行中的計時，請先使用 /start。`

---

### `/log`

**用途**：手動補記工時（不使用計時器）

**互動流程**：
1. 開啟 Modal，包含：
   - 下拉選單：選擇專案（RC / ASUS / BOT）
   - 文字欄位：工作內容（必填）
   - 數字欄位：時數（必填，如 `2.5`）
2. 送出後 Bot 回覆確認，並附上兩個按鈕：
   - **再新增一筆** → 重新開啟相同 Modal
   - **完成** → 結束
3. 條目加入當日 list：`{ project, content, hours, date }`

---

### `/summary`

**用途**：手動產生工時日報並 append 至 Canvas

**支援格式**：
- `/summary`：當天
- `/summary 2026-04-14`：指定單日
- `/summary 2026-04-01 2026-04-14`：指定區間
- `/summary this-week`：本週（週一～今天）
- `/summary last-week`：上週（週一～週日）

**互動流程**：
1. 解析日期參數，決定查詢區間
2. 從 `logs.json` 讀取該區間所有條目，依日期排序
3. 每天一個區塊，依專案分組，格式化輸出
4. 呼叫 Slack Canvas API，append 此段內容至該使用者的 Bot DM Canvas
5. Bot 回覆：`已將 2026-04-01 ～ 2026-04-14 紀錄存入 Canvas ✓`

**邊界條件**：
- 若指定區間無任何條目，Bot 提示：`該區間沒有工時紀錄。`
- 若日期格式錯誤，Bot 提示正確格式範例

---

### 排程自動寫入

**用途**：在工作時段自動寫入當日工時，不需手動執行 `/summary`

**排程時間**：09:00、14:00、19:00（週一至週五）

**觸發邏輯**：
1. 掃描 `logs.json`，找出今天有紀錄的所有 DM 頻道（userId）
2. 對每個頻道：
   - **有資料** → 自動將今日條目 append 至該頻道的 Canvas
   - **無資料** → 在該 DM 頻道傳送提醒：`你今天還沒有工時紀錄，記得填寫！`
3. 週六、週日不觸發

**說明**：「有在用的頻道」定義為 `logs.json` 中存在該 userId 的任意歷史資料。

## Canvas 格式範例

### 單日
```
## 2026-04-14
### Richart:
- 開會討論需求 1.5 小時
- 撰寫規格 2 小時

### ASUS:
- UI 調整 1 小時
```

### 區間（每天一個區塊）
```
## 2026-04-13
### BOT:
- 串接 API 3 小時

## 2026-04-14
### Richart:
- 開會討論需求 1.5 小時

### ASUS:
- UI 調整 1 小時
```

## 多人使用

- 所有記憶體資料以 `userId` 為 key 隔離
- 每位使用者在自己與 Bot 的 DM 頻道操作，互不影響
- Bot 安裝至 Workspace 後，所有成員皆可使用
